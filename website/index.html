ICEland

Your personal command-line tool for focused digital areas, notes, and time tracking.

What is ICEland?

ICEland is a Rust-based command-line tool that helps you separate your digital life into clear, focused areas such as work, learning, math, gaming, traveling, and trading. It creates structured environments with notes, links, and basic time tracking so you can stay organized and see where your time goes.

Who is this for?

  • Students who want to separate study, work, and personal time.
  • Developers who like clean, structured workflows.
  • Anyone who gets distracted by mixed files, notes, and browser activity.

What does ICEland do now?

With one tool, ICEland sets up and manages:

  • Dedicated folders for each area under ~/.iceland.
  • Area-specific tools (notes, flashcards, projects, analysis, etc.).
  • Separate browser profile spaces per area (simple profile folders, plus optional browser launch per area).
  • Useful links per area (Primuss, Wikipedia, games, Forex Factory, news, etc.).
  • A way to switch between areas instantly via CLI or a simple TUI (arrow keys).
  • A way to reset (“destroy”) browser data or notes per area.
  • A basic time log (sessions) per area in ~/.iceland/sessions.csv and a stats command to summarize it.

Installation

Download the ICEland Linux binary from the project’s GitHub releases or build it yourself with: 

cargo build --release

Then put the compiled binary somewhere in your $PATH (for example /usr/local/bin) and run it from your terminal.

Core Commands

Initialize areas

iceland init

Creates the base folder ~/.iceland and default areas: work, math, learning, gaming, traveling, trading. Each area gets its own subfolders (like notes, flashcards, browser_*) and some starter links.

Switch areas (classic CLI)

iceland switch learning
iceland switch gaming

Updates the current area stored in ~/.iceland_current, prints the area path, and shows any links.txt for that area. When you switch from one area to another, ICEland also writes a simple time entry for the previous area into ~/.iceland/sessions.csv (basic session log).

After switching, ICEland tries to launch a browser using the area name as profile (for example, firefox -P work for the work area). You can adapt this behaviour in the code to match your browser and OS.

Switch areas with a TUI (arrow keys)

iceland tui

Opens a simple terminal UI list of all default areas. Use the up/down arrow keys to select an area and press Enter. Internally this does the same as iceland switch <area> (including time logging and showing links).

Notes per area

iceland notes learning "Study Rust at 5pm"
iceland notes math "Do exercise 5"

Appends a line of text to ~/.iceland/<area>/notes/my_notes.txt. This keeps your notes clearly separated for each area.

Check current area

iceland status

Shows which area is currently active and the filesystem path for that area.

Reset browser or notes for an area

iceland destroy gaming browser
iceland destroy learning notes

Resets selected parts of an area:

  • browser: clears and recreates any browser_* directories inside that area.
  • notes: clears and recreates the notes folder for that area.

Time statistics per area

iceland stats

Reads the session log in ~/.iceland/sessions.csv and prints the total time (in seconds) per area, based on recorded entries. This is a simple approximation that you can later extend into full time tracking (start/stop, daily reports, etc.).

How ICEland helps you focus

When you switch to learning, you see Primuss, Wikipedia, and ChatGPT links. When you switch to gaming, you see simple web games. When you switch to trading, you see Forex Factory and space for analysis.

With the TUI and time stats, you can quickly select your current context and later review how much time you spent in each area.

Future & Pro Ideas

ICEland is currently free and open source. In the future, a “Pro” version could include:

  • Richer time tracking (real start/stop sessions, daily/weekly reports, exports).
  • Automatic, configurable browser launching per area (Chrome/Firefox/Edge profiles, work vs gaming windows).
  • Integration with task managers or calendars to create tasks or log time.
  • A more advanced TUI dashboard or even a small desktop GUI.
